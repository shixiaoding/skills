# study-examiner 使用教程

## 这个 Skill 是干什么的？

一句话：**你学完一份材料后，让 AI 像小老师一样一题一题考你；全部答完，再统一批改、讲解和补弱。**

它适合检验自己是否真的理解了内容，而不是只看过、觉得“好像懂了”。

## 安装

推荐安装到所用 Agent 的用户级 skills 目录：

```bash
# Codex
npx @helloxiaoding/ding-skills install study-examiner --agent codex

# Claude Code
npx @helloxiaoding/ding-skills install study-examiner --agent claude-code
```

Qoder CLI、QoderWork、覆盖已有 Skill 和手动复制安装方式见 [小鼎 Skills 安装手册](../installation.md)。

## 怎么用？

先提供已阅读的材料、文档或明确的主题范围，然后直接提出测验请求。

```text
我已阅读完成，请一题一答测验我，最后再统一讲解和补弱。
```

也可以这样说：

```text
当小老师考我这篇文章。
帮我检查一下我对 Spring 事务传播机制的吸收程度。
刚学完这章内容，给我来一轮快速测验。
```

如果没有提供材料，Skill 会先确认要考查的主题和范围，再开始出题；它不会自行假设你学过哪些内容。

## 测验过程中会发生什么？

默认完整测验为 8–12 题，先考基础理解，再逐步覆盖因果、边界、对比和真实场景应用。大部分题目是开放问答，选择题或判断题也会要求说明原因。

考试阶段每次只出一道题。你答完后，AI 只会中性记录并继续下一题，不会提前告诉你答对还是答错，也不会泄露答案。

如果想调整节奏，可以直接说：

```text
这次只考 5 题。
暂停考试，先讲解刚才的问题。
结束考试，现在开始复盘。
```

其中“暂停考试，先讲解”会切换出考试模式；否则，在考试中提出的概念问题会被记录，留到最终复盘时统一回答。

## 最终会得到什么？

全部答完后，AI 会统一给出：

- 大致吸收度；
- 已掌握内容与薄弱点；
- 逐题批改和简评；
- 考试中暂存的追问说明；
- 更通俗的薄弱点讲解；
- 是否需要进行 3–5 题精准补弱，或转向真实场景练习的建议。

默认只进行一轮完整测验，不会自动反复出题。只有存在关键误区或吸收度不足时，才会建议围绕薄弱点进行一轮小范围补弱。

## 常见问题

**Q：没有文章或笔记，也能用吗？**

可以，但需要说清楚主题和范围，例如“考我 Java 集合框架，重点是 `HashMap` 和并发集合”。范围越明确，题目越贴合你的学习目标。

**Q：为什么答完一题不马上告诉我对错？**

为了避免后续答案被提示影响。统一批改更能判断你在没有线索时真正掌握了什么。

**Q：能直接让 AI 讲解吗？**

可以。明确说“取消考试，切换为讲解模式”即可；这时不再遵循一题一答和延后讲解的规则。
