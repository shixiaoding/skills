# 🛠️ 我写了一个小技能，帮自己"学完东西能留下点什么"

> 这是我个人做的**第一个 Skill**，后面会持续更新。这个仓库是个人技能库，不是专门做资料收集的工具。

大家好，我是**奥利瓦**。

每次想了解一个东西，比如 **"Superpowers + OpenSpec 怎么结合使用"** ，流程都很固定：搜索 → 看文章 → 复制粘贴 → 关掉网页 → 过两周忘了。**下次又得重新搜一遍。**

我不想重复劳动。于是折腾了一个小东西——**Skills**，给 AI Agent 用的可复用技能包。

---

## 🎯 解决什么问题？

跑通一个**学习闭环**：

```
[提问] → [AI搜索] → [分类整理] → [生成文档] → [同步Notion] → [阅读+内化]  
↑***************下次增量更新***************↓
```

**把搜索过的东西留下来，变成自己的。**

---

## 📦 第一个技能：research-summary

支持四种使用场景：

| 场景 | 触发示例 | 做什么 |
| :--- | :--- | :--- |
| **首次了解** | `帮我收集 OpenSpec v3 资料` | 全量搜索 → 分类 → 生成文档 |
| **继续深入** | `基于上一版再补充案例` | 保留旧内容 → 增量补充 |
| **修改调整** | `改一下第3章表述` | 精准定位修改 |
| **只更新链接** | `后续只更新资料库` | 结构不动，只维护资料列表 |

---

## 🖼️ 真实案例

### 案例1：首次收集 —— 先生成骨架

> "帮我收集 Superpowers + OpenSpec 组合实践的资料，生成 md 并同步到 Notion"

**生成结果**（骨架）：  
👉 https://www.notion.so/Superpowers-OpenSpec-3679fce202f28157ae7aded0838e7dab

特点：框架清晰（定位→概念→路径→资料库→我的理解），内容待补。**价值：5分钟搭好架子。**

### 案例2：增量更新 —— 再填充内容

> "基于上一版，再补充 OpenSpec v3 的中文 CLI 资料"

**补充结果**（完整内容）：  
👉 https://www.notion.so/OpenSpec-v3-3679fce202f2815fbb59f7ed4b5df0ae

特点：完整翻译、使用说明、示例代码。**价值：在骨架上长出"血肉"。**

### 对比

| 维度 | 骨架 | 完整内容 |
| :--- | :--- | :--- |
| 完整度 | 框架清晰，内容待补 | 详实可用，有翻译和示例 |
| 耗时 | 5分钟 | 增量补充 |
| 价值 | 克服空白页恐惧 | 形成可用的知识资产 |

**我的习惯：先从骨架开始，再慢慢填充。**

---

## 🧠 核心目标：沉淀成自己的内容

注意第二个文档，它不只是资料堆砌：
- 中文翻译（读起来更快）
- 使用说明（可直接上手）
- 示例代码（可复制粘贴）

**这是我消化过的痕迹。**

分工：
- AI：收集 + 整理 + 粗加工
- 我：阅读 + 思考 + 内化

最终沉淀的是**我消化过的知识**，存在自己的 Notion 里。

## 💭 一点心得

AI 最大的价值不是"替我做"，而是 **"帮我省掉重复的体力活，让我专注在思考上"** 。

搜索、分类、整理这些事，AI 做得比我快。但"读懂了没有""有没有自己的观点"，还是得我自己来。

这个技能库还**非常早期**，是我的个人小项目。后续会持续更新（周报、复盘、知识沉淀等）。
