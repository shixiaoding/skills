# 资料收集总结 — 参考规范

## 一、资料库表格字段规范

| 字段 | 说明 |
|---|---|
| 分类 | 官方 / 大厂 / 社区 / GitHub / 论文 / 工具 |
| 资料名称 | 原始标题或项目名 |
| 来源 | 来源平台或组织 |
| 类型 | 官方文档 / 平台文章 / 社区文章 / GitHub 项目 / 论文 |
| 可信度 | A / B / C / D |
| 推荐程度 | 必看 / 推荐 / 选择看 / 待验证 |
| 主要价值 | 这条资料解决什么问题 |
| 链接 | 原文地址 |
| 可访问性 | 已确认 / 需人工确认 / 失效原链接 / 找不到正式链接 / 可能无效 |
| 是否原文 | 是 / 否 / 不确定 |
| 替代链接 | 原链接失效时填写 |
| 状态 | 已确认 / 已收录 / 待验证 / 待补链 |
| 备注 | 补充说明 |

### 可信度分级

| 可信度 | 类型 | 使用方式 |
|---|---|---|
| A | 官方文档 / 官方仓库 | 可作为主要依据 |
| B | 大厂技术文章 / 官方博客 / 论文 | 可作为重要参考 |
| C | 社区经验 / 个人博客 / 转载 | 作为经验补充 |
| D | 待验证 / 无法打开 / 来源不明 | 只归档，不作为结论依据 |

### 推荐程度

| 推荐程度 | 含义 |
|---|---|
| 必看 | 入门或主线必须读 |
| 推荐 | 强相关，值得看 |
| 选择看 | 有参考价值，但非必要 |
| 待验证 | 暂不确认有效性 |

## 二、工作流总览

```
识别用户意图
→ 判断工作模式
→ 收集资料
→ 整理资料表
→ 生成 Markdown
→ 判断版本策略
→ 判断是否请求 Notion 同步
→ （如已请求且服务可用）同步 Notion
→ 输出摘要
```

### 工作模式

| 模式 | 触发语义 | 动作 |
|---|---|---|
| 首次收集 | 帮我收集 / 整理 | 创建新文档 |
| 新版本 | v2 / v3 / 不覆盖 | 创建新版本 |
| 修改 | 修改当前文档 | 修改指定文档 |
| 资料库更新 | 这版确认了 | 只更新资料表 |
| 正式版合并 | 其他文档不要了 | 合并为主文档 |

## 四、Notion 同步规则（按需执行）

- 仅在用户明确要求同步且 Notion MCP 服务可用时执行。
- 默认创建新页面，不覆盖旧文档。
- 用户明确要求修改时才更新旧页面。
- 新版本标题格式：`主题 v1`、`主题 v2`...
- 修改模式只改指定章节。
- 资料库更新模式只改资料表，不改主结构。
- 已同步时输出 Notion 页面链接；未同步时说明原因，并输出修改/新增摘要和后续建议。

### 页面命名规则

| 场景 | 格式 |
|---|---|
| 首次页面 | `主题资料库` |
| 版本页面 | `主题资料库 v1` / `主题资料库 2026-05-21` |
| 正式版 | `主题资料库 v3 正式版` |

### 目录选择规则

1. 用户指定 → 使用用户指定目录。
2. 未指定 → 优先放到与主题最相关的知识库目录。
3. AI / 编程 / 工具类主题 → 放入 AI 工作台或 AI 知识卡片。
4. 不确定时 → 创建在当前相关页面下。

## 五、修改与更新区分

| 用户意图 | 含义 | 动作 |
|---|---|---|
| 修改 | 当前内容不符合预期 | 修改当前文档 |
| 新版本 | 基于旧版继续演进 | 新建版本 |
| 资料库更新 | 结构已确认，只更新资料 | 不改结构，只更新表格 |
| 定期更新 | 周期性补最新资料 | 只维护资料库 |
