# 团队级 Code Review Checklist

> 说明：
> - 本清单用于增量 Code Review
> - 只对本次变更相关代码进行检查
> - 若无法仅凭 diff 判断，请标记“需确认”

---

## 1. 代码规范与可读性

### 1.1 命名规范
- 类名 / 方法名 / 变量名是否清晰可读
- 是否存在随意缩写、歧义命名、拼音混用
- 常量是否统一抽取并命名明确

### 1.2 可读性
- 核心逻辑是否易理解
- 是否存在超长方法 / 多层嵌套 / 条件爆炸
- 复杂逻辑是否有必要注释

### 1.3 单一职责
- 类/函数职责是否过重
- 是否存在明显重复代码可抽取

---

## 2. 接口契约与兼容性

### 2.1 接口兼容
- API / RPC 返回结构是否破坏向后兼容
- DTO 字段删除、改名、类型变化是否会影响旧调用方
- 枚举新增/修改是否考虑兼容

### 2.2 接口规范
- 是否有成功/失败状态表达
- 错误类型是否区分“系统错误”和“业务错误”
- 敏感字段是否直接返回

---

## 3. 基础业务逻辑

### 3.1 状态流转
- 是否存在非法状态跳转
- 是否绕过状态机/前置校验
- 是否存在重复提交导致的状态错乱

### 3.2 业务规则
- 优惠、折扣、库存、支付、退款、审批等逻辑是否明显有误
- 是否覆盖关键边界条件
- 是否考虑空值、零值、重复值、最大最小值

### 3.3 幂等
- 创建、支付、回调、消费、同步、导入等写操作是否具备幂等保护
- 是否依赖唯一键、状态前置校验、token、防重表等机制

---

## 4. 数据一致性、事务与并发

### 4.1 事务边界
- 是否事务过大
- 是否在事务中执行远程调用 / 长耗时逻辑
- rollback 条件是否完整

### 4.2 数据一致性
- 多表更新是否有一致性保障
- 本地事务 + MQ / 事件是否存在先后顺序风险
- 分布式一致性是否至少有补偿方案

### 4.3 并发控制
- 并发写场景是否有锁/乐观锁/唯一约束/幂等控制
- 是否存在并发下重复写、超卖、覆盖更新等风险

---

## 5. 性能与资源管理

### 5.1 SQL / 查询
- 是否命中索引
- 是否存在 `SELECT *`
- 是否缺少 limit / 分页
- 是否存在明显 N+1 查询
- join / 子查询是否复杂且无性能依据

### 5.2 循环与集合
- 是否循环内查库 / 调 RPC / 调缓存
- 是否存在死循环风险
- 是否有明显不合理的大对象/大集合处理

### 5.3 资源释放
- 连接、文件、流、线程池、协程上下文是否正确释放
- 是否存在资源泄漏风险

### 5.4 缓存
- 热点数据是否有合理缓存
- 是否存在缓存击穿 / 雪崩 / 脏读风险
- 缓存与数据库一致性策略是否清楚

---

## 6. 安全与合规

### 6.1 输入安全
- 是否存在 SQL 注入风险
- 是否存在命令注入、路径注入、模板注入风险
- 是否缺少基础入参校验

### 6.2 数据安全
- 敏感数据是否明文存储或明文日志打印
- 日志是否脱敏
- 导出/下载能力是否存在越权或泄露风险

### 6.3 权限控制
- 是否校验登录态、角色、租户、数据归属
- 是否存在通过修改 ID 越权访问他人数据的风险

---

## 7. 异常处理与健壮性

### 7.1 外部依赖
- 调第三方 / 下游服务是否设置超时
- 是否有重试、熔断、降级或兜底
- 是否区分强依赖与弱依赖

### 7.2 异常处理
- 是否吞异常
- 是否 catch 过大范围但无有效处理
- 是否错误地把异常当流程控制

### 7.3 日志
- 关键异常是否带业务上下文
- error / warn / info 级别是否使用合理
- 是否缺少关键链路日志

---

## 8. 配置治理与发布回滚

### 8.1 配置治理
- URL、key、timeout、阈值等是否硬编码
- 是否应使用配置中心 / 环境配置
- 默认值是否安全

### 8.2 灰度
- 新逻辑是否支持灰度开关 / 白名单 / 分批放量
- 开关是否可动态控制

### 8.3 回滚预案
- 新老逻辑是否兼容
- DB 变更是否有回滚思路
- 配置切换是否可快速恢复

---

## 9. 可观测性

### 9.1 日志可追踪
- 关键链路是否能通过业务ID / traceId 追踪
- 是否有创建、更新、失败等关键日志

### 9.2 指标与告警
- 核心失败场景是否可监控
- 高频错误、超时、重试是否可观测

---

## 10. 语言专项检查

### 10.1 Java
- 金额是否使用 BigDecimal
- 是否误用 `double/float`
- 是否误用 `Executors`
- 是否存在包装类型拆箱 NPE 风险
- MyBatis 是否误用 `${}`
- POJO / DTO / DO 命名与使用是否规范
- 是否符合团队 Java 研发规范中的关键要求

### 10.2 PHP
- SQL 是否使用预处理
- 金额是否使用 `bcmath`
- 是否存在弱类型比较导致逻辑问题
- 是否存在数组 key / null / empty 误判
- 是否存在未捕获异常或致命错误风险

### 10.3 Go
- goroutine 是否泄漏
- channel 是否存在阻塞/未关闭/误用
- context 是否透传
- error 是否显式处理
- defer 是否合理

### 10.4 Python
- 是否存在可变默认参数问题
- 是否存在异常吞没
- SQL / ORM 是否安全
- 是否有类型与空值处理风险
- 并发/异步场景是否存在资源未释放
