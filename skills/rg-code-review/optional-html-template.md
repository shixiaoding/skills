# HTML 报告生成要求（可选）

当用户明确要求生成 HTML 报告时，输出一个简洁、可读、适合团队内部查看的静态页面。

## 文件名
```text
code-review-report-YYYYMMDD-HHMMSS.html
```

## 页面结构
1. 报告标题
2. 生成时间
3. 概览卡片
   - 文件数
   - 问题总数
   - P1/P2/P3 数量
4. 总体结论
5. 问题列表
   - P1 红色
   - P2 橙色
   - P3 灰色
   - 每条问题需展示“建议方案：怎么改（含示例）”
6. 做得好的地方
7. 人工确认项

## 样式要求
- 简洁，不追求炫技
- 白底 + 卡片布局
- 明确分级颜色
- 移动端可读
- 代码片段使用等宽字体
- 避免超长复杂 CSS
- “建议方案：怎么改（含示例）”建议使用独立信息块展示，保证可读性

## 原则
- HTML 仅作为报告载体，不要把主要精力耗在 UI
- 内容准确性优先于视觉效果
